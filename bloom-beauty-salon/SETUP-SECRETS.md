# Налаштування секретів — ДІМ КЕРАТИНУ

Всі приватні дані (пароль, токен бота, TurboSMS) зберігаються виключно
всередині Google Apps Script у розділі **Properties** — вони не потрапляють
у код і не видимі через браузер.

Займає ~5 хвилин. Виконується ОДИН РАЗ.

---

## Крок 1 — Відкрити Apps Script

1. Зайди на [script.google.com](https://script.google.com)
2. Відкрий проєкт **ДІМ КЕРАТИНУ**

---

## Крок 2 — Відкрити Script Properties

1. У лівому меню натисни шестерню ⚙️ **Project Settings**
2. Прокрути вниз до розділу **Script Properties**
3. Натисни **Add script property**

---

## Крок 3 — Додати кожен секрет окремо

Додай по черзі 5 властивостей (назву пиши точно як вказано):

| Property name    | Value (що вставити)                              |
|------------------|--------------------------------------------------|
| `ADMIN_PASSWORD` | Твій пароль для адмін-панелі (наприклад: `MySecurePass2026!`) |
| `BOT_TOKEN`      | Токен бота від @BotFather (наприклад: `7412345678:AAHxxx...`) |
| `BOT_USERNAME`   | Username бота без @ (наприклад: `dim_keratin_bot`) |
| `RECIPIENTS`     | JSON-масив з отримувачами (дивись нижче)         |
| `TURBOSMS_TOKEN` | Токен з кабінету turbosms.ua (або залиш порожнім якщо не використовуєш) |
| `TURBOSMS_SENDER`| Ім'я відправника SMS (наприклад: `DimKeratin`)   |

### Формат RECIPIENTS

Скопіюй і заміни chat_id своїм (як отримати — дивись TELEGRAM-SETUP.md):

```
[{"name":"Ангеліна","chat_id":"914794563"}]
```

Якщо потрібні двоє:
```
[{"name":"Ангеліна","chat_id":"111222333"},{"name":"Софія","chat_id":"444555666"}]
```

---

## Крок 4 — Зберегти і передеплоїти

1. Натисни **Save script properties**
2. Поверніться до редактора
3. Натисни **Deploy → Manage deployments**
4. Обери поточний деплой → **Edit** (олівець)
5. У полі Version: обери **"New version"**
6. Натисни **Deploy**
7. Скопіюй новий URL деплою
8. Встав його у `js/config.js` у рядок `GOOGLE_SCRIPT_URL = '...'`

---

## Крок 5 — Перевірити

1. Відкрий сайт → заповни тестову форму запису
2. Ангеліна має отримати Telegram-повідомлення
3. Клієнт (якщо вказав @username) — підтвердження
4. Зайди на `/admin.html` → введи пароль → маєш побачити запис

---

## Що змінилось (для розуміння)

| Було (небезпечно)                        | Стало (безпечно)                          |
|------------------------------------------|-------------------------------------------|
| Токен бота у `config.js` (публічний файл) | Токен у Apps Script Properties (сервер)  |
| Пароль у `config.js`                      | Пароль у Properties, перевірка на сервері|
| Telegram-запити з браузера клієнта        | Telegram-запити тільки з сервера          |
| Статичний ключ `DIM_KERATIN_ADMIN_2026`   | Одноразовий токен сесії (8 годин)         |
| Пароль перевірявся в браузері (обхід за 1 сек) | Перевірка на сервері, затримка при помилці |
| Усі дані клієнтів читались за фіксованим URL | Доступ тільки після серверної авторизації |

---

## Важливо після цього налаштування

- **Не додавай** BOT_TOKEN, ADMIN_PASSWORD та інші секрети назад у `config.js`
- **Не комітай** `.env` або файли з паролями у git
- Якщо забув пароль — зайди в Script Properties і зміни `ADMIN_PASSWORD`
- Якщо бот скомпрометований — відклич токен через @BotFather і встав новий у Properties
